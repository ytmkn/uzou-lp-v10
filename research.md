## 2. 受賞サイト分析から得た10の黄金パターン

詳細: `docs/award-winners-analysis.md`

1. **業種の常識を裏切る** — AdTech LP = ダーク+テック感 の常識を壊す
2. **見出しフォントで企業DNAを語る** — Noto Sans JP以外の個性的フォント
3. **2色+白の色設計** — 補色関係の2色で視覚的張力を作る
4. **動きを不均一にする** — マーキー/GIF/カルーセルを使い分け
5. **セクション名を固有名詞化する** — 「サービス」→独自ワード
6. **border-radiusに意味を持たせる** — 0px/200px/5pxの意図
7. **コンテンツから逆算してデザインする** — テンプレに流し込まない
8. **企業語を避けたコピーライティング** — 短い、具体的、唯一無二
9. **ローディングをブランド接点にする**
10. **FVは「説明」ではなく「体験」にする** — 事業説明はスクロール後

## 3. リファレンスサイト ブランディング分析（6サイト）

### 3-1. StartPass（BtoB SaaS — UZOUに最も近い）
- **配色**: 黒背景 + `#BFEF00`（ネオングリーン）+ `#EA1E25`（赤アクセント）
- **フォント**: Zen Kaku Gothic New（見出し）+ Archivo（本文）
- **h2: 160px!** 超巨大タイポグラフィ
- **ボタン**: ネオングリーン、border-radius: 32px（pill）
- **学び**: BtoB SaaSでも黒背景+蛍光色で攻められる

### 3-2. KAGAMI（EdTech SaaS）
- **配色**: `#0086CC`（ブルー）+ `#76A4BA`（淡色アクセント）
- **フォント**: Tazugane Gothic（プレミアム和文フォント）
- **ボタン**: 非対称border-radius（8px/40px/8px/40px）— ユニーク!
- **シャドウ**: `rgba(52,165,253,0.15) 4px 4px 40px` — 有彩色シャドウ
- **学び**: 非対称border-radiusとカラーシャドウで独自性

### 3-3. gmlabs（テック企業）
- **配色**: `#FFFF99`（ペールイエロー）+ `#007CFF`（ブルー）on `#E6E1DE`（ウォームベージュ）

- **ボタン**: border-radius: 0px（直角!）
- **学び**: 温かいベージュ背景 + シャープなボタンの対比

### 3-4. IMA Inc.（クリエイティブ企業）
- **配色**: `#DA1626`（大胆な赤）

- **学び**: 本文サイズを大胆に大きくする勇気

### 3-5. ライデン（ブランディング会社）

- **フォント**: Helvetica Now Display + ゴシックMB101 H（超プレミアム和文）
- **ボタン**: border-radius: 0px
- **学び**: エレクトリックブルー+ペールイエローの対比

### 3-6. スペースマーケットテクノロジー

## 4. v11で適用すべき設計原則

### 4-1. レイアウト革命
- [ ] 中央揃え見出しは最大2セクションに限定
- [ ] 左揃え+右コンテンツ、フルブリード、非対称グリッド（60/40）を混在
- [ ] 1セクションは見出しなし（ビジュアルから直接入る）
- [ ] Scale（数値）はフルブリードダーク背景で画面いっぱいに

### 4-2. タイポグラフィ革命
- [ ] 見出しフォントをAdTech/データの個性を語るものに変更
- [ ] セクションごとに見出しサイズを変える（核心=72px、補足=18px）
- [ ] 1箇所にセリフ体/イタリック/等幅フォントを混ぜる
- [ ] letter-spacingを0.1em（ラベル）〜-0.03em（見出し）で振る

### 4-3. 色彩革命
- [ ] 2色+白の明快なパレット
- [ ] CTAに補色/対比色のアクセントカラーを使う
- [ ] 有彩色シャドウ（グレーシャドウ禁止）
- [ ] 背景に白/オフホワイト交互ではなく、中盤にダークセクションを挿入

### 4-4. アニメーション革命
- [ ] fadeInUpは最大2セクション。他は左/右/スケール/回転/即表示
- [ ] マーキー（横流れ）を1箇所導入
- [ ] パララックスを1-2箇所
- [ ] ホバーを最低3パターン（ワイプ/ボーダー色変化/3D tilt）

### 4-5. 余白・密度革命
- [ ] 余白レンジを40px〜240pxに拡大（現在の80-120→5倍の振れ幅）
- [ ] 情報密度に波を作る: 低→高→低→高
- [ ] Trust帯は32px以下に圧縮
- [ ] Solutionは200px以上の大余白で「ここが核心」と示す

### 4-6. コンテンツ革命
- [ ] Features: プロダクトUIのモックアップ画像/Lottieアニメーション
- [ ] Trust: 実ロゴ（最低8社。ないならリアルなモック）
- [ ] Voices: 顔写真/アバター + 企業ロゴ
- [ ] Hero: 実データを感じさせるビジュアル（数字の奔流、データストリーム）
- [ ] セクション名を固有名詞化（「サービス」「実績」を使わない）
