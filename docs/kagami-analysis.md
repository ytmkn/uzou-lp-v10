### 背景色の切り替えパターン
- **基本は白背景**（`rgb(255, 255, 255)` = `#FFFFFF`）
- メインの特徴紹介エリアで**水色グラデーション背景**に切り替え（top: 3110〜9098）

- **ハードカットが基本** — グラデーションフェードやSVG波形は使っていない
- ロゴスクロール帯がセクション間の視覚的区切りとして機能
- 課題提起→KAGAMI紹介の境界で、背景が白→水色に切り替わる際に`border-radius: 100px 0px 0px`の丸角が出現（上辺左角のみ100px丸角で接続）

### padding-top/bottomのリズム
- **ヒーロー**: pt 80px / pb 53px（ヘッダー分のoffset込み）
- **導入事例**: 余白は内部要素で管理
- **課題提起**: pb 86px
- **KAGAMI紹介**: pt 140px（他セクションより広い）
- 全体的に**均一な余白設計**で、意図的な緩急は弱い

## 5. アニメーション・インタラクション

### STUDIO CMSのスクロールアニメーション
- **`.sd` クラス** + **`.appear` クラス** でスクロール連動のフェードインを実装
- 未表示時: `transform: matrix(1, 0, 0, 1, 0, 20)` + `opacity: 0` （translateY: 20px）
- 表示後: `transform: matrix(1, 0, 0, 1, 0, 0)` + `opacity: 1`
- **移動距離は 20px** — 控えめ
- ページ遷移: `cubic-bezier(0.4, 0.4, 0, 1)` で 0.6s（入場）/ 0.3s（退場）

### ロゴスクロール帯
- `@keyframes scrollLeft`: `translateX(0%)` → `translateX(-100%)` / 80s linear infinite
- `@keyframes scrollRight`: `translateX(-100%)` → `translateX(0%)` / 80s linear infinite
- 2列構成で上下が逆方向にスクロール

### 導入事例カルーセル
- 7枚のカードが自動切り替え
- `_playing` クラスでアクティブ管理
- `__ariaHidden` でアクセシビリティ対応

### トランジション
- **共通easing**: `cubic-bezier(0.4, 0.4, 0, 1)` — STUDIO CMS標準
- **ボタン**: `transition: 0.4s cubic-bezier(0.4, 0.4, 0, 1)`
- **ページ遷移**: `0.6s cubic-bezier(0.4, 0.4, 0, 1)`（入場）

### ホバーエフェクト
- STUDIO CMSの制約上、DevToolsからの動的ホバースタイル抽出は困難
- ボタンには `transition: 0.4s` が設定されている

## 6. ビジュアルアセット

### イラスト
- **フラットイラスト**をメインビジュアルとして使用（isometric寄りのアイソメトリックビュー）
- ヒーロー: 人物×3 + PC画面UIのイラスト
- 各セクション: SVGインライン画像（`svg%3E`がsrcに含まれる）で描かれたアイコン・チャート・UI画面のイラスト
- **写真は導入事例の人物のみ** — それ以外は全てイラスト

### 画像フォーマット
- **WebP**を基本使用（ロゴ、ヒーローイラストの一部）
- **SVG**が大量使用（セクション内のすべてのイラスト・図解）
- 各SVGは STUDIO CMS のレンダリングで `<img src="data:image/svg+xml,...">` として埋め込み

### ロゴスクロール帯
- 導入企業ロゴを横一列に配置
- 2列構成: 上行（4736x144）+ 下行（5168x144 / 4736x144）
- WebP画像として一枚のスプライト画像に統合

### 装飾的要素
- **セクション背景のイラスト装飾**: 水色系のグラデーション背景上に半透明のUIイラスト
- **ガラスモーフィズム的カード**: `rgba(255,255,255,0.5)〜0.7` の半透明白背景
- **有彩色グロー**: CTAボタンに `rgba(87,186,253,0.4) 0px 2-4px 50px` の大きな光彩

## 7. 独自性・テンプレ感排除のポイント

### このサイトの強み

#### (A) 非対称角丸システム
最も特徴的な設計要素。`border-radius: 8px 40px` のような対角線上で異なる角丸値を使うことで:
- テンプレートにはない手作り感
- 方向性の示唆（右上が丸い→進行方向を暗示）
- 視覚的な多様性（同じカードでも角の向きで変化）

#### (B) 有彩色シャドウの徹底
全てのシャドウが `rgba(95,131,175,0.1)` や `rgba(87,186,253,0.x)` のブルー系。
グレーシャドウ `rgba(0,0,0,x)` は一切なし。これにより:
- ブランドカラーとの統一感
- 画面全体に色の温度が通る
- テンプレートの「無機質さ」が消える

#### (C) ガラスモーフィズム（グラスモーフィズム）
- `rgba(255,255,255,0.5)〜0.7` の半透明カード
- `inset 4px 4px 0 0 #fff` の内側白影
- 水色背景の上に重ねることで奥行き
- ただしbackdrop-filterは未使用（パフォーマンス配慮か）

#### (D) テキストグラデーション
- セクションタイトルに `background: linear-gradient(90deg, #1959B4, #5BBDFF)` + `background-clip: text`
- テキスト自体がブランドカラーを体現

#### (E) CTAの光彩
- `box-shadow: rgba(87,186,253,0.4) 0px 2px 50px 0px` — blur 50pxの大きな光彩
- ボタンが「発光」しているように見える
- 視線誘導として機能

### このサイトの弱み（UZOUで改善すべき点）

#### (1) letter-spacingが未調整
全要素 `normal`。大見出しで `-0.02em`、小テキストで `+0.02em` の調整があれば格段に上がる。

#### (2) 余白のリズムが均一
セクション間のpadding変化に意図的な緩急がない。情報密度の変化に合わせた余白設計が弱い。

#### (3) 写真の活用が少ない
導入事例以外は全てイラスト。実際のプロダクトUIのスクリーンショットがあるとリアリティが増す。

#### (4) アニメーションが単調
全セクション同じ `translateY(20px) + opacity:0→1` のフェードイン。セクションごとのバリエーション（左から、右から、スケール等）がない。

#### (5) セクション境界の処理が甘い
基本ハードカット。グラデーションフェードやSVG装飾で接続すれば流れが生まれる。

#### (6) ダークモード/ダークセクション未使用
全体が白+水色の明るいトーンで、視覚的コントラストのダイナミクスが弱い。

## 8. UZOU LPへの技法適用提案

### 採用すべき技法

| 技法 | KAGAMIでの実装 | UZOU向けカスタマイズ |
|---|---|---|
| **有彩色シャドウ** | `rgba(95,131,175,0.1)` | UZOUのブランドカラーに合わせた有彩色影に置換 |
| **非対称角丸** | `8px 40px` パターン | 採用。ただしUZOUのトーンに合わせて値は調整 |
| **CTA光彩** | `blur 50px, opacity 0.4` | 採用。CTAボタンに光彩効果を適用 |
| **テキストグラデーション** | `linear-gradient + text-clip` | 見出しのアクセントに活用 |
| **ガラスモーフィズム** | `rgba(255,255,255,0.5-0.7)` | ダーク背景上なら `backdrop-filter: blur()` も追加して品質向上 |
| **ロゴスクロール帯** | `scrollLeft/scrollRight 80s` | 実績企業の信頼性演出に採用 |

### 採用しない/改善して使う技法

| 技法 | 理由 | UZOU向け改善 |
|---|---|---|
| letter-spacing: normal | 調整不足 | 大見出し -0.02em、本文 0、キャプション +0.02em に階層化 |
| 均一アニメーション | 単調 | セクションごとに異なるeasing・方向・stagger |
| 全面白背景 | コントラスト弱い | ダーク↔ライト交互のセクション設計 |
| STUDIO CMS cubic-bezier(0.4,0.4,0,1) | 独自性あるが少し直線的 | `cubic-bezier(0.22, 1, 0.36, 1)` をメインに |
| Tazugane Gothic一本 | 品質は高いが入手困難 | Noto Sans JP + Inter等のWeb Font組み合わせ |

### 全体所感
KAGAMIは STUDIO CMS で構築された BtoB SaaS LP として堅実な仕上がり。特に非対称角丸と有彩色シャドウはテンプレート感を排除するのに効果的に機能している。一方で、タイポグラフィの字間調整、アニメーションの多様性、セクション間の視覚的ダイナミクスには改善の余地がある。

UZOUのLPでは、KAGAMIの「有彩色シャドウ」「非対称角丸」「CTA光彩」を基盤技法として採り入れつつ、letter-spacingの階層化、ダーク/ライト交互のセクション設計、多様なスクロールアニメーション、backdrop-filter併用のグラスモーフィズムで差別化を図るべき。
