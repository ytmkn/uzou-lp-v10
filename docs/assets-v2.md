> 作成日: 2026-02-28

> AI生成/外部ライブラリ: 不使用

---

## 1. アセット戦略方針

全ビジュアルアセットを**CSS / SVG / Canvas 2D**で自作する。理由:
1. ネットワークグラフ、SVG図解、ノイズテクスチャはコードで高品質に実現可能
2. KAGAMIの技法（有彩色シャドウ、非対称角丸、ガラスモーフィズム、CTA光彩）は全てCSS値
3. 写真は不使用（導入事例は仮テキスト、ロゴはプレースホルダー）
4. 外部依存ゼロでパフォーマンス最適化

| アセット | 役割 | 調達方法 | リファレンス |

**Canvas ネットワークグラフ仕様**:
- 配置: 右45%の領域
- パーティクル数: 60-80（パフォーマンス制約: 上限100）
- パーティクル色: `#497E92`（ティールブルー）、半径2-4px
- 接続線: 距離閾値120px以内で描画。色: `rgba(73,126,146,0.15)`、幅: 0.5px
- ハイライトノード: ランダムに3-5個を`#E07B5A`（コーラル）に点灯（「見つけた」瞬間を表現）
- マウス追従: マウス位置にradial-gradient `rgba(73,126,146,0.06)` を重ねる
- フレームレート: 60fps、requestAnimationFrame使用
- `prefers-reduced-motion` 時: パーティクル停止、静的表示

**SVGノイズテクスチャ仕様**:
```svg
<filter id="noise"><feTurbulence baseFrequency="0.65" numOctaves="3" stitchTiles="stitch"/></filter>
```
- opacity: 0.03（白背景上で知覚できるギリギリの弱さ）
- 全画面に適用、position: fixed

### 2-4. SIFT（核心価値セクション）

| アセット | 役割 | 調達方法 | リファレンス |
|---|---|---|---|
| インタラクティブ接続図解 | UZOUの仕組みを1枚で理解させる | インラインSVG + CSS transition | KARTE |
| 信号パルスアニメーション | 「データが流れている」ことを動きで伝える | CSS animation | — |
| ガラスモーフィズムカード | 図解を格納するカード。奥行き | CSS | KAGAMI |

**SVG接続図解仕様**:
- ノード構成: 広告主側3つ（左）→ UZOU AI 1つ（中央、大）→ メディア側3つ（右）
- 中央ノード: 80x80px、border-radius: 200px、背景`rgba(255,255,255,0.15)` + `backdrop-filter: blur(8px)`
- 左右ノード: 48x48px
- 接続線: SVG `<path>`。stroke: `rgba(245,247,248,0.2)`、stroke-width: 1.5
- 信号パルス: 小さな円(r=3)がパスに沿って移動。CSS animation `offset-path`
- ホバー時: ノード背景が`#E07B5A`（コーラル）に変化、接続線がコーラルに太く

**ガラスモーフィズムカード仕様**（KAGAMI技法翻訳）:
- background: `rgba(255,255,255,0.12)`
- backdrop-filter: `blur(16px)`
- border: `1px solid rgba(255,255,255,0.15)`
- border-radius: `8px 32px`（非対称角丸。KAGAMI `8px 40px`の翻訳）
- box-shadow: `rgba(73,126,146,0.08) 0 4px 24px`（有彩色シャドウ）

KAGAMI実測値との対応:
| KAGAMI | UZOU |
|---|---|
| `rgba(255,255,255,0.5-0.7)` | `rgba(255,255,255,0.12)`（ティール背景上なので低透過でも効果あり） |
| `inset 4px 4px 0 0 #fff` | 不使用（ティール背景では不適切） |
| backdrop-filter未使用 | `blur(16px)`追加（KAGAMI改善点） |
