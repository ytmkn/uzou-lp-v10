# ビジュアルアセット戦略: UZOU LP v10

## アセット戦略方針

v10「Signal Pulse」のアセットは**全て自作（CSS/SVG/JS）**で完成させる。
理由:
1. 回路パターン・パルスアニメーションはCSS/SVGで高品質に実現可能
2. AI生成画像が必要な要素がない（v9のクリスタル画像のような写実的要素がない）
3. 外部依存を排除し、パフォーマンスを最適化
4. コンセプトに完全にフィットするカスタムアセットが作れる

## セクションごとのアセット要件

### 1. Hero — 信号の覚醒

| アセット | 役割 | 調達方法 | リファレンス |
|---|---|---|---|
| 回路パターン背景 SVG | 世界観を一瞬で伝える | 自作 SVG | Stripe のメッシュグラデーション背景の「精密さ」を回路で再現 |
| パルスアニメーション | 「生きているプラットフォーム」感 | CSS animation | Raycast のグロー明滅のテンポ感 |
| マウス追従グラデーション | インタラクティブ感 | CSS + JS | Plaid のマウス追従手法 |

**回路パターン SVG の仕様**:
- 直線（水平/垂直）の接続線。斜め線は使わない（基板的精密さ）
- 接続点にノード（円形、r=3-5px）
- 線色: rgba(139, 192, 202, 0.15)（ライトティールの15%透過）
- ノード色: rgba(139, 192, 202, 0.4)（パルス時は0.8まで上昇）
- パターンサイズ: 200x200px のタイルを repeat
- ノードは6-8個/タイル。ランダムに配置しない、グリッドに沿わせる

### 2. Trust — 信号の到達先

| アセット | 役割 | 調達方法 |
|---|---|---|
| メディアロゴ | 社会的証明 | ユーザー提供依頼 |
| マーキー実装 | 信号の流れの表現 | CSS animation (marquee) |

**ユーザーへの依頼事項**:
- UZOUに接続しているメディアのロゴ画像（SVG推奨、PNG可）
- 最低8メディア分。不足する場合はプレースホルダーで仮実装

### 3. Problem — ノイズの検出

| アセット | 役割 | 調達方法 |
|---|---|---|
| 課題アイコン（6種） | 各課題の視覚的区別 | 自作 SVG アイコン |

**SVGアイコンの仕様**:
- 線画スタイル（stroke-width: 1.5-2px）
- サイズ: 24x24px viewBox
- 色: ディープティール #2B4954
- スタイル: 回路的な直線ベース（丸みは最小限）
- 6種: シールド（ブランド毀損）、チャート下降（CPA頭打ち）、レポート（煩雑）、コイン（低単価）、ユーザー離脱（UX損害）、時計（手が回らない）

### 4. Solution — 信号経路の設計図

| アセット | 役割 | 調達方法 | リファレンス |
|---|---|---|---|
| 接続図解 SVG | UZOUの仕組みを直感的に理解させる | 自作 SVG + JS | KARTE のインタラクティブ SVG 図解 |

**接続図解の仕様**:
- 3つのメインノード: 広告主（左）→ UZOU AI（中央）→ メディア（右）
- 中央のUZOUノードは他より大きく（60-80px）、ティールのグロー付き
- 左右のノードは40-50px
- 接続線: SVG path（直線 + 角丸の折れ線）
- スクロール連動: 線が左から右へ stroke-dashoffset で描画される
- ノード点灯: 線が到達したノードからティール色に点灯
- 中央ノードの下に「CTR/CVR/視認性 総合分析」のラベル

### 5. Scale — 信号の増幅

| アセット | 役割 | 調達方法 |
|---|---|---|
| 数値カード上辺のパルスライン | カードの回路ボーダー | CSS (border-top + box-shadow animation) |

**パルスラインの仕様**:
- border-top: 2px solid #34626F
- box-shadow: 0 -2px 8px rgba(139, 192, 202, 0.3) で微グロー
- ホバー時: box-shadow が 0 -2px 16px rgba(139, 192, 202, 0.5) に拡大

### 6. Features — 信号処理の3つの核

| アセット | 役割 | 調達方法 |
|---|---|---|
| 機能概念図 SVG（3種） | 各機能の直感的理解 | 自作 SVG |
| 回路的装飾線 | 機能間の接続感 | CSS (border + pseudo element) |

**概念図 SVG の仕様**:
- Feature 1（AI最適化）: ニューラルネットワーク的なノード接続図。入力→処理→出力の3層
- Feature 2（メディアネットワーク）: 中央ハブから放射状に伸びる接続線（スター型トポロジ）
- Feature 3（運用サポート）: 人物アイコン + ダッシュボード的な枠線
- サイズ: 各 240-320px 幅
- 色: ティール系のモノトーン（#2B4954, #34626F, #8BC0CA の3色のみ）
- スタイル: 線画ベース、fill最小限

### 7. Voices — 信号の反響

| アセット | 役割 | 調達方法 |
|---|---|---|
| 引用符アイコン | テスティモニアルの視覚記号 | 自作 SVG（"" マーク） |
| 会社ロゴ | 発言者の所属を示す | ユーザー提供依頼（テスティモニアルが仮なら不要） |

### 8. Flow — 接続プロトコル

| アセット | 役割 | 調達方法 |
|---|---|---|
| ステップアイコン（4種） | 各ステップの視覚的識別 | 自作 SVG |
| ステップ接続線 | 回路的なステップ間の接続 | SVG line + stroke-dashoffset animation |

**ステップアイコンの仕様**:
- 4種: メール（お問い合わせ）、ヘッドセット（ヒアリング）、設定ギア（アカウント開設）、ロケット（配信開始）
- サイズ: 32x32px viewBox
- 色: ミドルティール #34626F
- 接続線: 各ノード間を水平線でつなぐ。スクロール連動で左から順に描画

### 9. FAQ

| アセット | 役割 | 調達方法 |
|---|---|---|
| 開閉アイコン | アコーディオンの状態表示 | CSS（+/- or chevron を transform: rotate で切替）|

### 10. Final CTA — 信号の発信

| アセット | 役割 | 調達方法 |
|---|---|---|
| 全回路パルスアニメーション | 緊急感とインパクト | Hero の回路パターン SVG を再利用 + 全ノード同時パルス animation |
| CTAボタンのパルスグロー | ボタンへの注目誘導 | CSS animation (box-shadow pulse) |

**全回路パルスの仕様**:
- Hero と同じ回路パターン SVG を再利用
- ただし、ノードのパルス animation を全ノード同時に + 周期を短く（1.5秒）
- 接続線の opacity も 0.15 → 0.3 に上昇（全体が明るくなる）
- 視覚的に「全回路が活性化した」印象
