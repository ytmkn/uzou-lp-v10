## 1. コンセプトワード

### **「Sift — 有象無象を、見極める知性」**

> UZOUの名前は「有象無象（うぞうむぞう）」から生まれた。
> 膨大な情報の海の中から、本当に価値のあるものだけを見つけ出し、届ける。
> ノイズの中からシグナルを。混沌の中から秩序を。
>
> 「Sift」とは、ふるいにかけること。
> テクノロジーは声高に主張しない。
> 水面の下で深く、静かに、知的に働く。
> 表に見えるのは、精度の高い結果だけ。
>
> ティールブルーの深さは、その「水面下の知性」を表す。

## 2. 世界観

| 要素 | 定義 |
|---|---|
| 空間 | 深い水底から水面を見上げた景色。光が水を通して揺らぎ、無数のデータの粒子が漂う。その中からUZOUのアルゴリズムが必要なものだけを選り分け、光のある水面へ送り出す。静謐で広大。テクノロジーの気配はあるが冷たくない。水が持つ知性——流れを読み、最適な道を選ぶ |
| 時間 | 水の流れの時間。急がず、しかし確実に進む。潮の満ち引きのような、大きなリズムの中で成果が積み上がっていく。「瞬時の判断」と「長期の信頼」が共存する時間感覚 |
| 質感 | 磨かれたガラスと流水。透明感があり、奥行きが見える。表面はなめらかだが、光を受けると微細な揺らぎが現れる。KAGAMIのガラスモーフィズム的半透明カード(`rgba(255,255,255,0.5-0.7)`)を、ティールブルーの世界に翻訳する |
| 温度 | 冷たすぎない涼やかさ。ティールブルーが持つ知的な冷涼感に、コーラルの差し色が人間的な温もりを加える。信頼できる相手と向かい合ったときの、適度な距離感と安心感 |

## 3. デザイン原則

1. **Depth（奥行き）** — 全ての要素に前景/中景/背景の層を持たせる。フラットな面はゼロ。有彩色シャドウ（KAGAMI技法: グレー影禁止、全影を`rgba(73,126,146,x)`や`rgba(224,123,90,x)`で）、半透明レイヤー、ブラー、非対称角丸で「水深」を感じさせる

2. **Rhythm（波のリズム）** — セクション間の余白、タイポグラフィのサイズ、色の明暗を波のように変化させる。均一なリズムはテンプレートの証拠（v10の診断で指摘された中間8セクションのpadding均一問題を根絶）。paddingは4段階（48px / 100px / 120px / 160px）で意図的な緩急を設計する

3. **Precision（精度）** — 「かしこい」とは精度が高いこと。余白は8pxグリッドに厳密に乗せる。フォントサイズは論理的な比率で決定する。letter-spacingを大文字ラベル`+0.12em`〜和文見出し`-0.03em`の5段階に階層化する（KAGAMI弱点: 全要素`normal`を克服）

### モーションディレクション

- **動きの人格**: 水中を浮かぶような減速運動。急がず、しかし確実に停止する
- **入場イージング**: `cubic-bezier(0.16, 1, 0.3, 1)` — 水面に浮かび上がるような減速
- **ホバーイージング**: `cubic-bezier(0.25, 0.46, 0.45, 0.94)` — 即座に反応、自然に減速
- **マーキー**: `linear` — 潮流の安定感
- **KAGAMI参照**: `cubic-bezier(0.4, 0.4, 0, 1)` はSTUDIO CMS独自。UZOUではより有機的なカーブを使用

## 7. 非対称角丸システム（KAGAMI技法のUZOU翻訳）

KAGAMIの`8px 40px`パターンをUZOUの「精度」コンセプトに合わせて再設計。

| パターン名 | border-radius値 | 使用箇所 | 意味 |
|---|---|---|---|
| **右上流れ** | `8px 32px` | 機能カード、引用カード（奥行きの方向性） | 水が右上に流れるような方向性。コンテンツの「進行方向」を示唆 |
| **左上流れ** | `32px 8px` | 課題カード（対比配置時） | 右上流れの鏡像。対称性を崩す |
| **シャープ** | `0px` | FAQ項目、区切り線 | データのエッジ。確信。精密さ |
| **水滴** | `6px` | CTAボタン、入力フィールド | わずかな丸みで親しみ |
| **球体** | `200px` | ステップ番号丸、アバター、タグ | 完全な球。流れの中の特別な存在 |

KAGAMI実測値との対応:
- KAGAMI `8px 40px` → UZOU `8px 32px`（40pxを32pxに抑制。精密さを維持）
- KAGAMI `40px 8px` → UZOU `32px 8px`
- KAGAMI `100px 0 0` → 不採用（セクション角丸はUZOUの精密トーンに合わない）
- KAGAMI `48px 48px 48px 4px` → 不採用（吹き出し系はUZOUに不要）

## 8. コンセプト数値制約

| 要素 | レンジ | コンセプトからの根拠 |
|---|---|---|
| セクション余白 | 48px〜200px | 「波のリズム」。浅い水深(48px: TRUSTED BY帯)と深い水底(160px: SIFT/THREE SIGNALS前後)で呼吸を作る |
| アニメーション速度 | 200ms〜1200ms | 「精度の速度」。ホバー=水面の即時反応(200ms)、入場=水中を浮かぶ速度(500-600ms)、countUp=確信の到達(1200ms) |
| フォントウェイト差 | 見出し500-800 vs 本文400 | 見出しの重さがPrecisionを伝える。数値のweight 800は「揺るぎない事実」 |
| 色の明度差 | WCAG AAA推奨（7:1以上） | ダーク(`#0D2B33`)上のテキスト(`#E8F0F3`)は14.2:1。精度原則に基づく |
| 影のblur範囲 | 4px〜60px | 小blur(4px)=浅い水深。大blur(60px)=深い水底からの光。KAGAMI CTAのblur 50pxに対応 |
| border-radius | 0px / 6px / 8px / 32px / 200px | 非対称角丸システム。5値に限定（KAGAMI: 6パターンから精選） |
