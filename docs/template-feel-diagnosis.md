# UZOU LP v10 テンプレ感診断レポート

## 総括

UZOU LP v10は技術的には堅実に作られている。アクセシビリティ対応（aria属性、skip link、prefers-reduced-motion）、構造化データ、scroll-driven animations対応など、基盤品質は高い。しかし「受賞レベルのLP」と比較すると、全体に「丁寧に作られた汎用テンプレート」の域を出ていない。最大の問題は**構造・色・動き・空間のすべてが「均一」であること**。予測可能性が高すぎて、訪問者の感情を揺さぶる瞬間がない。

---

## 1. レイアウトの画一性

### 何が問題か

全10セクションが「中央揃えタイトル → 説明テキスト → コンテンツグリッド」の同一パターンで繰り返されている。

```css
/* 7つのセクションヘッダーが同一構造 */
.problem__header  { text-align: center; margin-bottom: 64px; }  /* L722 */
.solution__header { text-align: center; margin-bottom: 64px; }  /* L830 */
.scale__header    { text-align: center; margin-bottom: 64px; }  /* L937 */
.features__header { text-align: center; margin-bottom: 80px; }  /* L1035 */
.voices__header   { text-align: center; margin-bottom: 64px; }  /* L1148 */
.flow__header     { text-align: center; margin-bottom: 64px; }  /* L1255 */
.faq__header      { text-align: center; margin-bottom: 48px; }  /* L1393 */
```

7セクション中6セクションで `text-align: center; margin-bottom: 64px;` が完全一致。コンテナ幅も全セクション `max-width: 1120px; margin: 0 auto;` で固定。

スクロールするたびに「見出し中央 → コンテンツ」の同じリズムが繰り返され、脳が次の展開を予測できてしまう。「またこのパターンか」という認知の慣れが起き、内容に関係なく情報が素通りする。

### 受賞レベルのLPはどう違うか

- **フルブリード × コンテインド**を意図的に混在させる。ある場面では画面幅いっぱいにビジュアルを広げ、別の場面では極端に幅を絞って読ませる
- テキストを左寄せにするセクション、右寄せにするセクション、全画面スプリットにするセクションなど、**視線の動き自体をデザイン**する
- 「見出し → 説明 → コンテンツ」の3段構成を崩す。見出しなしでいきなりビジュアルから始まるセクション、見出しが画面いっぱいに広がるセクションなどを混ぜる
- 非対称グリッド（60/40分割、70/30分割）を活用し、均等分割の退屈さを回避する

### 具体的にどう変えるべきか

- Solutionセクションをフルスクリーンの没入体験に変える。見出しを排除し、スクロールに連動して接続図が画面全体で展開される体験にする
- Scaleセクションの数値を画面幅いっぱいに1つずつ横スクロールで見せるか、スクロール連動で1つずつ大きく出現させる
- Problemセクションのタイトルを左寄せにし、課題カードを右側にスタッキングするなど、左右非対称なレイアウトを導入
- 少なくとも3セクションは `max-width: 1120px` の制約から解放する
- Scaleの見出しは**小さめ（18-24px）のラベル的テキスト**にして、数字自体を主役にする

## 2. 色使いの単調さ

### 何が問題か

カラーパレット自体が5色（`#8BC0CA`, `#34626F`, `#2B4954`, `#1F353E`, `#040404`）+ 白系1色（`#F4F8F9`）で、**すべて同系統のティール/ダークティール**。アクセントカラーの `--color-pulse-amber: #F0A848` が定義されているのに**一度も使用されていない**。

```css
/* style.css L17-25: アンバーが定義だけあって未使用 */
--color-pulse-amber: #F0A848;
```

背景色の交互パターンが機械的:
```
Hero:      #1A2830（ダーク）
Trust:     #F4F8F9（ライト）
Problem:   #FFFFFF（ライト）
Solution:  #F4F8F9（ライト）
Scale:     #FFFFFF（ライト）
Features:  #F4F8F9（ライト）
Voices:    #FFFFFF（ライト）
Flow:      #F4F8F9（ライト）
FAQ:       #FFFFFF（ライト）
Final CTA: #1A2830（ダーク）
```

ライトセクションが8つ連続し、`#FFFFFF`と`#F4F8F9`の微差の交互になっている。この差はほとんどのモニターで知覚しにくく、全体が同一の明るいグレーにしか見えない。ダーク背景はHeroとFinal CTAの「サンドイッチ」で、これが典型的なテンプレート構成。

ドットパターン背景も3セクション（solution, features, flow）で全く同じ指定:
```css
/* 3セクションで完全同一 */
background-image: radial-gradient(circle, rgba(26, 40, 48, 0.02) 1px, transparent 1px);
background-size: 32px 32px;
```

CTAボタンも見出しも全てteal系で、視線の誘導先がない:
```css
.btn-primary { background: var(--color-mid-teal); }  /* 背景と同系色 */
```

### 受賞レベルのLPはどう違うか

- ページの途中でダークセクションを挿入し、「休憩」「転換」を演出する（例: 課題セクションをダークにして緊張感を出し、ソリューションで明るく転換する）
- 単色背景だけでなく、写真・テクスチャ・グラデーション・イラストを背景に使い分ける
- アクセントカラーを戦略的に使う。CTAボタン、重要数値、引用符など、注目させたい要素にだけ別系統の色を投入する
- 補色関係（teal × amber/orange）を活用して視覚的な緊張感を作り出す

### 具体的にどう変えるべきか

- **`--color-pulse-amber`をCTAボタンに適用する**。ティール一色の中にアンバーが入るだけで視覚的なリズムが生まれる。補色関係で強力に目を引く
- **Problemセクションの背景をダーク**（`#1A2830`系）にする。課題=暗い、ソリューション=明るいという色による物語を作る
- **Scaleセクションもダーク背景**にして、白い数字で映えるようにする。ページ中盤でリズムが変わる
- ライトセクションの交互パターン（`#FFFFFF` ↔ `#F4F8F9`）をやめる。代わりに背景テクスチャ、微妙なグラデーション、非対称のカラーブロックでセクション境界を作る
- セクション間のグラデーション遷移帯（`.hero__transition`, `.faq__transition`）が硬直的な線形グラデーションなので、曲線的・有機的な境界（SVG wave、clip-path）に変える
- Problem セクションの課題アイコンに赤~オレンジ系の警告色を使い、Solutionのアイコンと対比させる

## 3. タイポグラフィの平凡さ

### 何が問題か

全セクションの見出しが**完全に同一のCSS**で定義されている:

```css
/* 以下7つの見出しが全て同一指定（L727, L835, L942, L1040, L1153, L1260, L1398） */
font-family: var(--font-heading);
font-size: clamp(24px, 3.5vw, 40px);   /* 全て同じ */
font-weight: 700;                       /* 全て同じ */
line-height: 1.2;                       /* 全て同じ */
letter-spacing: -0.015em;               /* 全て同じ */
color: var(--color-deep-teal);          /* 全て同じ */
text-wrap: balance;                     /* 全て同じ */
```

- **フォント2種（Space Grotesk + Noto Sans JP）の使い分けが機械的**。見出し=Space Grotesk、本文=Noto Sans JPの二択で、それ以上の変化がない
- Heroのキャッチコピー (`clamp(32px, 5vw, 64px)`) とセクション見出しの間にサイズの「崖」がある。サブヘッド（20-24px）のような中間レベルが不在
- `letter-spacing` の変化が `-0.025em` ～ `0.04em` の狭い範囲で、タイポグラフィとしての表情が乏しい
- 行送り（line-height）が `1.2`（見出し）と `1.8`（本文）の2値のみ。キャプション、リード文、引用文で行送りを変える意図がない
- 本文テキストも全セクションで同一パターン: `font-size: 15-16px; font-weight: 400; line-height: 1.8; color: rgba(4, 4, 4, 0.7);`

どのセクションも同じ重要度に見え、情報のヒエラルキーが存在しない。「全部同じだから全部読まなくていい」と脳が判断してしまう。

### 受賞レベルのLPはどう違うか

- **見出しサイズに3-4段階の明確なヒエラルキー**を設ける。例: ヒーロー(72px) → 核心H2(56px) → 通常H2(36px) → ラベル(14px)
- 1つの書体ファミリー内でウェイト変化を大胆に使う（Light 300 ↔ Bold 700 ↔ Black 900）
- キーフレーズに大きなletter-spacing（0.1em以上）を適用し、ラベル的な存在感を出す
- テキストブロックのmax-widthを60ch前後に制限し、1行あたりの文字数を最適化する
- 引用文は明確にサイズやスタイルを変えて差別化する
- 色も `--color-deep-teal` 一色ではなく、セクションごとに微妙に変えるか、アクセントカラーの見出しを1箇所入れる

### 具体的にどう変えるべきか

- セクション見出しのサイズを均一にせず、最も重要な「UZOUが、つなぐ。」は56-72px級の特大に、「導入の流れ」「FAQ」は32px級に差をつける
- ヒーロータイトルのfont-weightを900にし、letter-spacingを-0.04emまで詰める。または逆にlight weightで大サイズにして空気感を出す
- 「Feature 01」「Step 01」などのラベルに `text-transform: uppercase; letter-spacing: 0.15em` を適用し、ラベル感を強調
- 本文の `max-width` を `560px`→`60ch`（約600px）に変更し、読みやすさの観点から最適化
- 引用テキスト（`.voices__quote-text`）のフォントスタイルを差別化（例: font-sizeを18px、font-weight: 400にし、行間をやや広めの2.0にする）
- コピーの「。」で終わる体言止めの力強さを、太いウェイト + 大きなサイズで強調

## 4. アニメーションの定型化

### 何が問題か

全セクションのアニメーションが`translateY(20px) + opacity: 0 → translateY(0) + opacity: 1`の**完全に同一のフェードアップ**。

```css
/* 以下の全クラスが同一アニメーション */
.hero__title      { transform: translateY(20px); }  /* L427 */
.hero__sub        { transform: translateY(20px); }  /* L444 */
.hero__cta-group  { transform: translateY(20px); }  /* L458 */
.problem__item    { transform: translateY(20px); }  /* L770 */
.scale__card      { transform: translateY(20px); }  /* L969 */
.features__item   { transform: translateY(20px); }  /* L1064 */
.voices__card     { transform: translateY(20px); }  /* L1180 */
.flow__step       { transform: translateY(20px); }  /* L1286 */
.final-cta__title { transform: translateY(20px); }  /* L1597 */
.final-cta__sub   { transform: translateY(20px); }  /* L1618 */

/* scroll-driven fallback も同じ */
[data-reveal]     { transform: translateY(20px); }  /* L1723, L1745 */
```

script.jsでも**7つの初期化関数がすべて同一ロジック**:
```js
initRevealAnimations()    // L159: threshold 0.15, fadeInUp
initProblemStagger()      // L183: threshold 0.15, fadeInUp
initScaleSection()        // L262: threshold 0.15, fadeInUp
initFeaturesAnimation()   // L329: threshold 0.15, fadeInUp
initVoicesAnimation()     // L350: threshold 0.15, fadeInUp
initFlowAnimation()       // L371: threshold 0.3, fadeInUp + stagger
initFinalCtaAnimation()   // L488: threshold 0.3, fadeInUp
```

唯一の差分はstaggerのdelay値（80ms or 120ms）のみ。Solutionの線描画だけが異なるが、setTimeoutによる固定タイミングのシーケンシャル点灯で、スクロール量との連動がない。

スクロールするたびに「下から浮き上がる」が繰り返されると、最初は良くても3セクション目あたりで「またか」と感じる。アニメーションが装飾でしかなく、情報を伝える手段になっていない。

### 受賞レベルのLPはどう違うか

- **セクションごとにアニメーションの「性格」が異なる**。数値は回転しながら出現、カードは手前に倒れ込むように立ち上がる、テキストは1文字ずつ現れるなど
- スクロール速度に応じたパララックス。要素ごとに異なる速度で動くことで奥行きが生まれる
- ホバーアニメーションが「translateY(-2px)」だけでなく、カードの傾き（perspective + rotateX）、内部要素のずれ、光の反射シミュレーションなど多様
- GSAP ScrollTriggerやScroll-driven Animationsを活用し、スクロール位置と1:1で連動する「スクラブアニメーション」を実装
- ページ全体を通して「物語の流れ」がある。課題提示時は要素が重く沈むように出現し、ソリューション提示時は軽やかに浮き上がるなど

### 具体的にどう変えるべきか

- **フェードアップの多用をやめる**。各セクションに固有のモーションを:
  - Problem: 要素が左右から「押し込まれる」ように出現（translateX）
  - Scale: 数値がカウントアップに加えてスケール変化（小→大、CSS maskまたはclip-pathで数字の上昇を表現）
  - Features: テキストブロックとビジュアルが逆方向からスライドインして出会う
  - Voices: カードが「トランプを配る」ように回転+移動で順番に配置される（CSS perspective + rotateY）
  - Flow: ステップ間の接続線がスクロールに同期して描画される（現状の固定タイミングを廃止）
- Heroのキャッチコピーに文字単位のスタガーアニメーションを導入（split text animation）
- Solution図解をスクロール駆動にする。setTimeoutベースを廃止し、scroll progressに連動させる
- ノードパルスにランダム性を加える（固定の6パターンdelayではなく、インラインstyleでランダムdelayを設定）
- ホバーエフェクトも多様化: CTAは背景色ワイプ、カードは微かな3D tilt、ナビリンクは下線スライドイン

## 5. 装飾の表面的さ

### 何が問題か

「回路パターン」というデザインモチーフが選ばれているが、その実装が**単なるSVGパターンの繰り返し配置**に留まっている:

```html
<!-- index.html L140-153: 200x200の固定パターンタイル -->
<pattern id="circuit" width="200" height="200" patternUnits="userSpaceOnUse">
  <!-- 水平線と垂直線の直交グリッドのみ -->
  <line x1="0" y1="40" x2="80" y2="40" stroke="rgba(139,192,202,0.12)" .../>
  <!-- ...同じパターンの繰り返し... -->
</pattern>
```

- 同一のパターン定義（`#circuit`, `#circuit-light`, `#circuit-final`）が3回出現。違いはstrokeのrgba値のみで、視覚的にはほぼ同じ
- `.circuit-node` のパルスアニメーションが「opacity 0.25→0.7 + scale 1→1.33」の単純な明滅。データの流れ、信号の伝搬といった「意味のある動き」がない
- ノイズテクスチャ（`#noise-filter`）が `opacity: 0.035` で実質見えない。存在意義が薄い
- SVG概念図（ニューラルネットワーク、スター型トポロジ、ダッシュボード）が**ワイヤーフレーム品質**。円と線だけで構成され、塗り・グラデーション・テクスチャがない

```html
<!-- index.html L401-436: Feature 1 のビジュアル - SVGの線と丸だけ -->
<svg viewBox="0 0 320 240">
  <circle cx="60" cy="60" r="8" stroke="#2B4954" stroke-width="1.5"/>
  <line x1="68" y1="60" x2="152" y2="45" stroke="#34626F" opacity="0.4"/>
  <!-- ...同じ構造の繰り返し... -->
</svg>
```

### 受賞レベルのLPはどう違うか

- 装飾がコンテンツの意味と連動する。「接続」を表現するなら、スクロールに応じてノード間にデータが流れるアニメーションを見せる
- SVGイラストレーションが単なるアイコンではなく、カスタムイラストとして作り込まれている。線画でもウェイトの変化、角の丸み、要素の密度に意図がある
- 背景テクスチャが「あるかないか分からない」ではなく、ブランドの質感として知覚できるレベルで存在する
- WebGLやCanvasを使った動的ビジュアル（パーティクル、フローフィールド、メッシュグラデーション）が没入感を作る
- プロダクトの実際のスクリーンショット（モックアップ）をビジュアルとして活用する

### 具体的にどう変えるべきか

- 回路パターンのSVGを、直交グリッドから**有機的なネットワークグラフ**に変える。ノード間の線が曲線で、ノードのサイズに変化がある
- ノードパルスを「ランダムな明滅」から「ノードA→ノードBへ信号が伝搬する」アニメーションに変える。CSSだけでは限界があるのでCanvasまたはWebGLで実装
- Features セクションのSVG概念図をリッチにする。fill-gradientの適用、ストロークのウェイト変化、ラベルテキストの追加。あるいは概念図の代わりに**実際のダッシュボードのモックアップスクリーンショット**を配置
- ノイズテクスチャのopacityを0.035→0.08程度に引き上げるか、グレインテクスチャをCSS `background-image: url(grain.png)` で直接適用する
- Trust セクションに**実際のメディアロゴ**を掲載する。「Media A」「Media B」のダミーテキストは「本当に導入企業があるのか」という疑念を生む

## 6. コンポーネントの既視感

### 何が問題か

- **数値カード（Scale）とテスティモニアルカード（Voices）が同一のスタイル構造**:

```css
/* Scale カード: L959 */
.scale__card {
  background: #FFFFFF;
  border: 1px solid rgba(43, 73, 84, 0.08);
  border-radius: var(--radius-lg);  /* 8px */
  padding: 32px 24px;
  box-shadow: var(--shadow-sm);
}

/* Voices カード: L1170 - 背景・ボーダー・角丸・影が完全一致 */
.voices__card {
  background: #FFFFFF;
  border: 1px solid rgba(43, 73, 84, 0.08);
  border-radius: var(--radius-lg);  /* 8px */
  padding: 32px;
  box-shadow: var(--shadow-sm);
}
```

- ホバーも同一パターン（`translateY(-2px)` + shadow強化）:
```css
.scale__card:hover  { transform: translateY(-2px); box-shadow: ...; }  /* L982 */
.voices__card:hover { transform: translateY(-2px); box-shadow: ...; }  /* L1193 */
.btn-primary:hover  { transform: translateY(-2px); box-shadow: ...; }  /* L518 */
.btn-secondary:hover { transform: translateY(-2px); box-shadow: ...; } /* L590 */
```

- **FAQアコーディオン**: +/-アイコンによる開閉は最も典型的なパターン。視覚的差別化ゼロ
- **ロゴマーキー（Trust）**: CSSアニメーションの横スクロールマーキーは2020年以降のLP定番。hover時のpauseも含めて100%テンプレート通り
- **フロー4ステップ**: 円形ノード → 接続線 → 次のノードの水平配置は無数に見てきたパターン
- **CTAボタン**: `translateY(-2px)` のhover + パルスグロー(`cta-pulse` keyframes) の組み合わせはSaaSテンプレートの定型表現

### 受賞レベルのLPはどう違うか

- 汎用コンポーネントの「形」を崩す。カードなら角丸を取り去って片側だけ丸くする、影ではなくborderの色変化でエレベーションを表現するなど
- テスティモニアルはテキストだけでなく、動画、手書き風署名、実写写真を組み合わせてリアリティを出す
- FAQをアコーディオンにせず、チャットUI風にするか、質問カードをクリックするとフリップして裏面に回答が表示されるなど
- フローは直線的な水平並びではなく、曲線のタイムライン、スクロール連動のステップバイステップ体験にする
- ホバーエフェクトの種類を最低3パターンは用意する

### 具体的にどう変えるべきか

- **Scale**: カードをやめて、数値を画面幅いっぱいの大文字で表示。ダーク背景+白数字で、スクロールに応じて1つずつフォーカスされる体験にする。または数値の背景にリングチャートやプログレスバーを組み合わせる
- **Voices**: カードの均一な3カラムをやめる。1つ目を大きく（幅2カラム分）表示し、残り2つを小さくスタックするなど、引用の重要度でサイズ差をつける。顔写真やイラストアバターを追加する
- **FAQ**: アコーディオンの+/-アイコンを番号表示（Q1, Q2...）に変え、開閉時にアイコンがモーフィングする。あるいはアコーディオンをやめて、左に質問リスト・右に回答表示のスプリットレイアウトにする
- **Trust**: ロゴにhoverしたときにそのメディアの簡単な紹介がツールチップで出るなどのインタラクションを追加
- **Flow**: 水平4ステップを縦のタイムラインに変更。スクロールに応じてステップがアクティブになり、各ステップの詳細がサイドに展開されるインタラクティブ体験にする
- **ホバー**: CTAは背景色がスライドで切り替わるワイプ + scale(1.02)、カードはボーダーカラーのアニメーション変化、ナビリンクは下線スライドイン

## 7. 空間設計の貧弱さ

### 何が問題か

- **paddingが機械的**: ほぼ全セクションが `padding: 80px 24px` または `padding: 100-120px 24px` で、セクション間の「呼吸」に変化がない

```css
.trust     { padding: 48px 24px; }
.problem   { padding: 80px 24px 64px; }
.solution  { padding: 120px 24px 100px; }
.scale     { padding: 80px 24px; }
.features  { padding: 120px 24px 100px; }
.voices    { padding: 80px 24px 64px; }
.flow      { padding: 100px 24px 80px; }
.faq       { padding: 100px 24px 80px; }
.final-cta { padding: 120px 24px; }
```

パターンは実質3種類: `80px`, `100px`, `120px`。差がわずか20px刻みで、スクロール時に「呼吸」が感じられない。

- **margin-bottomの均一性**: `.xxx__header` がすべて `margin-bottom: 48-80px` の範囲で固定。コンテンツの密度に関係なく同じ余白
- **gap値の単調さ**: グリッドのgapが `24px`, `32px`, `48px` の3パターンのみで、要素間の関係性（親密さ/独立性）を余白で表現できていない
- **垂直リズムの乱れ**: 8pxグリッドは定義されているが、余白が8の倍数に揃っていない箇所がある（margin-top: 12px, padding: 20px 0 など）
- **ネガティブスペースの不在**: すべてのセクションが「コンテンツを詰める」方向で設計されており、意図的に「何もない空間」を作って呼吸させる設計がない
- **セクション境界が全て1pxの直線border**: 全セクションに `border: 1px solid rgba(43, 73, 84, 0.08)` が付き、Excelの罫線のような事務的印象

### 受賞レベルのLPはどう違うか

- セクション間の余白が一定ではなく、**ストーリーの転換点で余白が大きくなる**（例: 課題→解決策の間は200px以上の余白で「転換」を演出）
- 1つのセクション内でも、要素間の距離で「関係性の近さ」を視覚化する（ゲシュタルトの近接の法則の意識的活用）
- フルブリード画像やビジュアルの後に、意図的に200px以上の「無」を配置して余韻を作る
- テキストブロックの左右に大きなマージンを取り、「白い額縁」の中にコンテンツを置くことで高級感を出す
- セクション境界をボーダーではなく、背景色の変化・SVGウェーブ・clip-pathで有機的に処理する

### 具体的にどう変えるべきか

- Problem→Solutionの間に意図的に160-200pxの余白を設け、「課題の重さ」と「解決への転換」を空間で表現する
- **重要セクション（Solution, Features）は上下200px以上**の大胆な余白で「ここが核心」と示す
- **Trust（ロゴ帯）は32px程度に圧縮**。軽いセクションとして扱う
- **Scale は余白なしでフルブリードダーク背景**にしてインパクトを出す
- 各セクションの `max-width: 1120px` を一部で `max-width: 800px` に絞り、コンテンツ密度に変化をつける
- 8px基本グリッドに厳密に揃える。12px, 20pxなどの非グリッド値を16px, 24pxに修正
- **ボーダーを全て削除**し、背景色の変化だけでセクションを区切る。一部は clip-path で斜めや曲線のカットを使う
- 情報密度のリズムを意図的に作る: 低密度（Trust） → 高密度（Problem/Solution） → 低密度（Scale、数字だけ大きく） → 高密度（Features） → 低密度（Voices）

## 8. コピーとビジュアルの乖離

### 何が問題か

コピーには「届く」「つながる」「見つけ出す」「伴走する」などの動詞的・ストーリー的表現があるのに、デザインがそれを全く反映していない。

**ヒーローコピー「届く広告は、つながりから生まれる。」** に対して、ビジュアルは汎用的な回路パターン。「つながり」を表現するなら、点と点が実際に結ばれていく動的なビジュアルがあるべきだが、静的なタイル繰り返しパターンでは「つながり」が可視化されていない。

**「500を超えるメディアと直接接続」** と言っているのに、Trust セクションのロゴが8つのダミーテキスト（Media A～H）。500という数の圧倒感が全く伝わらない:
```html
<!-- index.html L183-199: 8つのダミーロゴ -->
<span class="trust__logo-text">Media A</span>
<span class="trust__logo-text">Media B</span>
<!-- ...Media Hまで... -->
```

**「AIが最適な配信先を瞬時に見つけ出し」** に対して、AIの「賢さ」「スピード」を体感させるビジュアルがない。FeaturesのニューラルネットワークSVGは静的で、「瞬時に見つけ出す」感がゼロ。

**Solution「UZOUが、つなぐ。」** の図解が「広告主→UZOU AI→メディア」の3ノード直線接続。これでは「つなぐ」の価値が「中間に入るだけ」に見え、むしろ価値の低さを感じさせてしまう:
```html
<!-- index.html L319-337: 3ノードの直線接続 -->
<circle cx="120" cy="100" r="40" .../>  <!-- 広告主 -->
<circle cx="400" cy="100" r="56" .../>  <!-- UZOU AI -->
<circle cx="680" cy="100" r="40" .../>  <!-- メディア -->
```

**Scale「数字が証明する、接続の力。」** のカウントアップアニメーションは一般的すぎて「力」を感じない。「力」を表現するなら大きなフォント、ダーク背景、インパクトのある演出が必要だが、おとなしい白背景カード。

**「専任チームが、伴走する。」** は人間的な温かみを含む言葉なのに、ビジュアルは冷たいダッシュボード図（index.html L483-502）。人物写真やイラスト、手書き風要素があるべき。

### 受賞レベルのLPはどう違うか

- コピーの中のキーワード（つながり、500+、瞬時、伴走）が**ビジュアルで同時に語られる**。テキストを読まなくても、ビジュアルだけで同じメッセージが伝わる
- 数値はただカウントアップするのではなく、その数字の「意味」を視覚化する（500メディア→500個の点が画面を埋め尽くす、35%改善→グラフが劇的に跳ね上がる）
- プロダクトのスクリーンショットやデモ動画を実際に見せて、「使うとどうなるか」を具体化する
- コピーの感情（危機感、安心感、期待感）がそのままビジュアルのトーン（暗い色、暖色、明るい色）に反映されている

### 具体的にどう変えるべきか

- **Hero**: 回路パターンの代わりに、Canvas/WebGLで動的なネットワークグラフを描画。ノードが常に新しい接続を形成し続けるアニメーションで「つながりが生まれる」を可視化する
- **Trust**: 8つのダミーロゴではなく、500+の小さなドットが画面を埋め尽くし、一部がロゴに変化するアニメーションで「500以上」の数を体感させる。あるいは実際のメディアロゴを可能な限り掲載する
- **Solution**: 3ノード直線を、広告主側に複数ノード（業種別）、メディア側に多数ノード（カテゴリ別）を配置した**多対多のネットワーク図**に変更。UZOU AIが中央で最適なマッチングを行う様子をアニメーションで表現する
- **Features AI**: 静的なニューラルネットワークSVGの代わりに、実際のダッシュボードのモックアップ（Figmaで作成）を配置。データが流れ、最適化されていく様子のマイクロアニメーションを付ける
- **Features 伴走**: ダッシュボード図に人物イラスト/写真を追加。手書き風アクセント、暖色の差し色で「人間的な温かみ」を表現する
- **Scale**: カウントアップと同時に、背景のネットワークグラフのノード数が増えていくなど、数値とビジュアルが連動する演出を加える。「数字が証明する、接続の力」→ 数字を画面いっぱいに巨大表示、ダーク背景、カウントアップ時にパーティクルが散る
- **Flow**: 各ステップにUIスクリーンショットのミニプレビューを添え、「実際の体験」を想起させる
