# リファレンス一覧: UZOU LP v10

## リファレンス選定方針

v10「Signal Pulse」コンセプトに合致するリファレンスを3つの階層で選定。
v9で参照した結晶・ポリゴン系サイトは除外し、回路・信号・精密デザイン系を中心に選定する。

---

## ページレベルリファレンス（全体トーン・世界観）

### 1. Stripe (stripe.com)
**参考理由**: AdTech/フィンテック同士。ダークヒーロー + ライトセクションの切替が自然で、テック感と信頼感を両立している。「回路的な精密さ」の世界観に最も近い。

#### このサイトが優れている理由
決済インフラという「信頼」が必須のBtoBプロダクトで、テクノロジーの精密さを視覚的に表現しながらも親しみやすさを失わない絶妙なバランスを実現している。

#### 抽出技法（ディレクションレベル）
| 要素 | 技法 | 印象・効果 |
|---|---|---|
| 背景 | メッシュグラデーションの微動アニメーション | 生きている基板のような有機感 |
| タイポ | 幾何学サンセリフの太い見出し + 薄い本文 | 精密さと読みやすさの共存 |
| アニメーション | セクション間のグラデーション遷移（120px帯） | 切れ目のない信号の流れ |
| 装飾 | インラインのプロダクトUIデモ | 技術力を「見せる」信頼構築 |

#### この案件への応用
- メッシュグラデーション → ヒーロー背景。ただし色はUZOUティール系に限定し、v9のような多面グラデーションではなく、暗い基板にティールの光が差す表現
- セクション間遷移 → 全セクション境界でグラデーション帯を使い、「信号の流れ」を表現

---

### 2. Linear (linear.app)
**参考理由**: ダークUI × プロダクト訴求の最高峰。アンビエントライトの使い方がSignal Pulseの「発光するノード」と同じ思想。

#### このサイトが優れている理由
純黒の空間に、最小限の光で最大限の情報と美しさを実現している。装飾が一切なく、プロダクトUIそのものが美しいという説得力。

#### 抽出技法（ディレクションレベル）
| 要素 | 技法 | 印象・効果 |
|---|---|---|
| 背景 | 純黒 + 周辺からの放射状グロー | 暗闇に浮かぶ精密機器の印象 |
| タイポ | 見出しのグラデーションテキスト（white→grey） | 奥行き感、高級感 |
| アニメーション | 極めて微細なスクロール連動（opacity変化のみ） | 過剰さの排除 |
| 装飾 | 微光ボーダー（1px、rgba白10%程度） | 要素の境界を光で示す |

#### この案件への応用
- アンビエントライト → ヒーローとFinal CTAのダークセクションで、ティール系の放射グローを控えめに配置
- 微光ボーダー → カード・セクション区切りで回路の信号線として機能する1px線

## セクションレベルリファレンス

### 3. Vercel (vercel.com) — Trust/Scale セクション参考
**参考理由**: ロゴクラウドと数値実績の見せ方がBtoB SaaSの最良例。ミニマルでありながら信頼感が強い。

#### 抽出技法（ディレクションレベル）
| 要素 | 技法 | 印象・効果 |
|---|---|---|
| ロゴ帯 | グレースケールロゴの横一列 + 数値バッジ | 控えめだが確実な社会的証明 |
| 数値 | 超大型数値 + 小さなラベル | 数字のインパクトが第一優先 |
| レイアウト | 左右対称の中央揃え | 安定感、信頼感 |

#### この案件への応用
- Trust: グレースケールロゴマーキー + 接続メディア数を大きく表示
- Scale: 4つの数値を横一列、数値は48-64px、ラベルは14-16px

### 4. SmartHR (smarthr.jp) — Features セクション参考
**参考理由**: 日本のBtoB SaaS LPで最もバランスが良い。機能紹介の交互レイアウトが明快。

#### 抽出技法（ディレクションレベル）
| 要素 | 技法 | 印象・効果 |
|---|---|---|
| 機能紹介 | 左テキスト-右ビジュアルの交互レイアウト | 情報の整理、飽きさせない |
| 数値ハイライト | 大きな数値 + マーカー装飾 | 具体性、説得力 |
| アニメーション | スタッガードフェードイン（80ms刻み） | 順序立てた情報提示 |

#### この案件への応用
- Features: 3機能を交互レイアウト。各機能にSVGの概念図
- アニメーション: セクション内の要素を80ms刻みでスタッガード

### 5. KARTE (karte.io) — Solution セクション参考
**参考理由**: プラットフォーム図解（広告主→プラットフォーム→メディア）の見せ方が参考になる。SVG接続図解がインタラクティブ。

#### 抽出技法（ディレクションレベル）
| 要素 | 技法 | 印象・効果 |
|---|---|---|
| 図解 | SVGノード接続図。ホバーでハイライト | プラットフォームの仕組みを直感的に理解 |
| 背景 | 薄い回路パターン | テック感 |
| アニメーション | ノードの順次点灯 | データの流れの可視化 |

#### この案件への応用
- Solution: 広告主ノード → UZOUノード → メディアノードの3段階SVG図解
- スクロール連動で接続線が描画され、ノードが順に発光

## 技法レベルリファレンス

### 6. Plaid (plaid.co.jp) — マウス追従グラデーション
**参考技法**: マウス位置にradial-gradientが追従する効果。ヒーローセクションで使用。
**応用**: ヒーローのダーク背景で、マウス位置にティールのグローが追従。「信号を追いかける」インタラクション。

### 7. Raycast (raycast.com) — パルスグロー効果
**参考技法**: 要素のボーダーが脈動するように光るグロー効果。
**応用**: CTAボタンのボーダーにパルスグロー。Final CTAセクションの全ノード同時パルス。

### 8. Tore Knabe (toreknabe.com) — CSS scroll-driven animations
**参考技法**: JSなしのスクロール連動アニメーション（animation-timeline: view()）
**応用**: 入場アニメーション全般をCSS scroll-driven animationsで実装（Safari用にIntersectionObserverフォールバック）
