### 重要な観察: 色の使い方の「プロらしさ」

1. **純粋な白黒を使わない**: テキスト色は `#000000` ではなく `#0F1419`（わずかに青みのあるダーク）。背景も `#FFFFFF` だが、半透明の白（`rgba(255,255,255,0.5)`, `0.7`）を重ねて奥行きを出す
2. **同一色相の濃淡だけで構成**: ブルー系のみで全ページを構成。暖色は一切使わない

4. **半透明レイヤーの重なり**: `rgba(255,255,255,0.5)`, `0.6`, `0.7`, `0.698` など微妙に異なる不透明度で「ガラス的」な奥行きを作る

## 4. レイアウトパターン: AI感を排除する技法

### 4.1 ヒーロー: 非対称2カラム

```
|←108px→|←─── 612px (50%) ───→|←───── 760px (62%) ─────→|
|        |  テキスト (左寄せ)     |  ビデオ (画面右端までブリード)  |
|        |  Y: 80 → 635          |  Y: -63 → 715               |
|        |  padding-top: 80px     |  上にはみ出し(-63px)          |
```

**ポイント**:
- ビデオが右端からはみ出す（`width: 760px`, ビューポート端まで = 1440-720 = 720px のはずが 760px）
- ビデオのY位置が `-63px`（上にはみ出す）。これにより「枠に収まらない動的さ」を演出
- テキスト部分には `padding-top: 67px` で垂直方向にも余白
- **50:62の比率** は黄金比に近い

### 4.2 カルーセル: 角丸100pxの背景で「ゾーン」を作る

```
| ← 全幅1440px → |
| ╭────────────────╮  ← border-radius: 100px 0px 0px (左上のみ)
| │  背景: #EBF8FD │  ← bg: rgb(235, 248, 253)
| │  カルーセルカード │
| ╰────────────────╯
```

- 背景の左上だけ `border-radius: 100px` で丸める
- カルーセルカード: `border-radius: 8px 40px` + `border: 10px solid rgba(255,255,255,0.8)`

### 4.3 課題カード: 2重ネストによる奥行き

```
外枠: border-radius: 40px 8px, bg: rgba(255,255,255,0.698), shadow: rgba(95,131,175,0.1) 0 0 8px
  └── 内枠: border-radius: 28px 8px, bg: rgba(255,255,255,0.7), shadow: (同上)
       └── コンテンツ（動画 + テキスト）
       └── 背景装飾: border-radius: 32px, bg: rgb(224, 242, 254)
```

**半透明カードの2重構造**: 外枠と内枠で微妙にopacityが異なり（0.698 vs 0.7）、ガラスモーフィズム的な奥行きを生む

### 4.4 「課題」セクション: テキストとグラフィックの非対称配置

**課題1: マンネリ化を解決する**
```
|←── 617px (50%) ──→|←── 515px (42%) ──→|
|  イラスト（左）       |  テキスト（右）       |
|  画像が中央まではみ出す |                     |
```

**課題2: やりっぱなしを解決する**
```
|←── 462px (38%) ──→|←── 636px (52%) ──→|
|  テキスト（左）       |  イラスト（右）       |
|                     |  画像が左にはみ出す   |
```

課題1と課題2で **テキストとグラフィックの左右が反転**。これが視覚的リズムを作る。

### 4.5 ステップカード: 3x2グリッド

```
|← 384px →|← 24px →|← 384px →|← 24px →|← 384px →|
| STEP 1   |         | STEP 2   |         | STEP 3   |
|          |         |          |         |          |
| 406px高   |         |          |         |          |
|← 384px →|← 24px →|← 384px →|← 24px →|← 384px →|
| STEP 4   |         | STEP 5   |         | STEP 6   |
```

- カードサイズ: 384 x 406px
- gap: 24px（実測: X座標の差 522-108-384 = 30px? → 要再確認）
- border-radius: `8px 32px`（非対称角丸が全カード統一）
- shadow: `rgba(95, 131, 175, 0.1) 0px 0px 8px`

### 4.6 フッターCTA: 2カード並列

```
|←── 600px ──→|← 24px →|←── 600px ──→|
| CONTACT      |         | DOWNLOAD     |
```

影の方向: `4px 4px` = 右下方向。ここだけ影の方向が他と異なる（他は `0px 0px` で均一）

## 5. 角丸の階層設計（最重要パターン）

### KAGAMIの「非対称角丸」パターン

KAGAMIのデザインで最も特徴的なのは **非対称角丸** の体系的な使用。

| パターン | CSS値 | 使用箇所 | 意味 |
|---|---|---|---|
| **カード（大）** | `8px 40px` | カルーセル、ステップカード、フッターCTA | 左上と右下が鋭角、右上と左下が丸い |
| **カード（課題）** | `40px 8px` | 課題カード（外枠） | 上記の逆。左上と右下が丸い |
| **カード（課題内）** | `28px 8px` | 課題カード（内枠） | 外枠より小さい角丸 |
| **吹き出し** | `48px 48px 48px 4px` | 説明吹き出し | 左下だけ鋭角 |
| **ナビ分割（左）** | `10px 4px 4px 10px` | お問い合わせボタン（左） | 左が丸く右が鋭角 |
| **ナビ分割（右）** | `4px 10px 10px 4px` | 資料DLボタン（右） | 右が丸く左が鋭角 |
| **ボタン** | `16px` | CTAボタン | 均一な角丸 |
| **ピル型** | `100px` | タグ、ラベル | 完全なピル |
| **コンテナ** | `32px` | CTA包含ボックス、カード背景 | 大きめの均一角丸 |
| **背景ゾーン** | `100px 0px 0px` | 事例セクション背景 | 左上だけ丸い |
| **ドットアイコン** | `8px` | セクション見出し横のドット | 小さな装飾 |

### AI生成LPとの決定的な違い

**AI生成LP**: `border-radius: 12px` を全要素に一律適用。どこを見ても同じ角丸。
**KAGAMI**: `8px 40px` という非対称パターンを基本とし、要素の役割に応じて系統的に変化させる。

## 6. 影の設計

### 影のパレット

| 用途 | CSS値 | 特徴 |
|---|---|---|
| **カード標準** | `rgba(95, 131, 175, 0.1) 0px 0px 8px 0px` | 無方向、blur 8px、青灰色の有彩色影 |
| **CTAボタン** | `rgba(87, 186, 253, 0.4) 0px 2px 50px 0px` | 下方向2px、blur 50px（巨大）、ブルーのグロー |
| **お問い合わせボタン** | `rgba(87, 186, 253, 0.4) 0px 4px 50px 0px` | 上記と同系統だがoffset大 |
| **フッターCTAカード** | `rgba(52, 165, 253, 0.15) 4px 4px 40px 0px` | 右下方向、blur 40px |
| **ターゲットカード** | `rgb(255, 255, 255) 4px 4px 0px 0px inset` | 白のinset影（内側の光） |
| **吹き出し** | `rgba(87, 186, 253, 0.2) 0px 4px 20px 0px` | 中程度のブルーグロー |
| **ロゴサムネイル** | `rgba(87, 186, 253, 0.2) 0px 0px 11px 0px` | 無方向、小さめのグロー |

### 重要な観察

1. **全ての影が有彩色**: `rgba(0,0,0,x)` を一切使わない。影色は全て青系（rgb(87-131, 131-186, 175-253)）
2. **CTAのblur 50px**: ボタンの周囲に巨大なブルーのグローを作り、CTAの存在感を高める
3. **影の opacity**: カード 0.1 → 吹き出し 0.2 → CTA 0.4 と、重要度に応じて濃くなる
4. **inset shadow**: ターゲットカードだけ `inset` で内側に白い光を当て、ガラス的な質感を強調

## 8. 非対称性・グリッドの「崩し」

### 8.1 ヒーロー: テキスト50% vs ビデオ62%

通常の2カラムは50:50。KAGAMIはテキスト部 612px vs ビデオ部 760px で **合計がコンテナ幅を超える**。ビデオが右端をはみ出すことで「規則正しいグリッドに収まっていない」印象を作る。

### 8.2 課題1・2でのテキスト/グラフィック反転

- 課題1: 左にグラフィック（617px, 50%）、右にテキスト（515px, 42%）
- 課題2: 左にテキスト（462px, 38%）、右にグラフィック（636px, 52%）

左右の比率も 50:42 → 38:52 と変化させている。

### 8.3 角丸の非対称

`8px 40px` = shorthand for `border-radius: 8px 40px 8px 40px` → 右上と左下が大きく丸い。これは通常のUI設計では見ない。KAGAMIの最大のデザイン識別子。

### 8.4 テキスト配置

**圧倒的に左揃え**（142回 vs 中央29回 vs 右0回）。中央揃えは限定的にしか使わない。

中央揃えの使用箇所:
- ターゲットカードの見出し（「社員の離職率が高く、対策したい」等）
- 吹き出しタグ（「質の高い研修を手間なく内製化！」等）
- KAGAMIおすすめカードの内部テキスト

**左揃え基調であること自体が「AI感」を排除する大きな要因**。AI生成LPは中央揃えを多用しがち。

## 9. 装飾要素

### セクション見出しのドットアイコン

全セクション見出しの左に統一的なドットアイコンが付く:

正方形の中に小さい正方形。この「角丸正方形 in 角丸正方形」がブランドの視覚識別子として機能。

2. 課題カード1内: 488 x 371px
3. 課題カード2内: 497 x 383px

## 10. インタラクション・モーション

### トランジション

全要素に統一的なトランジション:
```css
transition: 0.3s cubic-bezier(0.4, 0.4, 0, 1);
```

このイージングカーブ `cubic-bezier(0.4, 0.4, 0, 1)` は:
- 開始: 中程度の加速
- 終了: 急減速（弾むような停止）
- STUDIOのデフォルトeasing

## 11. 「AI感」を排除するデザインの構造的分析

### AI生成LPが陥りがちなパターン vs KAGAMIの手法

| # | AI生成LPの典型 | KAGAMIの手法 |
|---|---|---|
| 1 | **完全対称の2カラム** (50:50) | テキスト50% vs ビデオ62% の**はみ出す非対称** |
| 2 | **全セクション同じpadding** | 0px / 80px / 120px / 140px / 167px / 196px と**意図的な緩急** |
| 3 | **border-radius: 12px 一律** | `8px 40px` / `40px 8px` / `48px 48px 48px 4px` 等**非対称角丸の体系** |
| 4 | **rgba(0,0,0,0.1)の灰色影** | `rgba(87,186,253,0.4)` blur 50px の**ブランドカラーグロー影** |
| 5 | **中央揃えテキスト基調** | **左揃え 142回 vs 中央 29回**。圧倒的に左揃え |
| 6 | **セクションごとに完結する色** | 全ページが**ブルーの単色相濃淡**のみで構成 |
| 7 | **カードは1重構造** | **2重ネスト**（外枠+内枠）で半透明レイヤーの奥行き |
| 8 | **全要素が枠内に収まる** | ビデオが上に-63px、右にはみ出す。**意図的なオーバーフロー** |
| 9 | **装飾要素がない or 汎用的** | 「角丸正方形 in 角丸正方形」のドットアイコンが**ブランド識別子** |
| 10 | **画像はセクション幅に収まる** | 全幅画像（1440px）がセクション区切りに使われ、**物理的なリズム切り替え** |
| 11 | **見出しサイズが段階的** (40→32→24→18) | **18px → 40px → 60px と大胆にジャンプ**。中間サイズを省略 |
| 12 | **行間が一律 1.5-1.6** | 本文 **line-height: 2.2** で「呼吸する」余白 |
| 13 | **テキストとグラフィックが常に同じ配置** | 課題1（左グラフィック/右テキスト）→ 課題2（左テキスト/右グラフィック）で**反転** |
| 14 | **背景色の変化が「切り替え」** | **グラデーション的に色が変化**（白→薄青→青→白）。境界が曖昧 |
| 15 | **アニメーションが全セクション同一** | ロゴマーキーのみ80s infinite。他は出現時の0.3s transition |

### 最も重要な3つの原則

#### 原則1: 非対称角丸 `8px 40px` というブランド記号

KAGAMIの最大の特徴は、**全てのカード・コンテナに非対称な角丸を使っている**こと。`8px 40px` は「右上と左下が大きく丸く、左上と右下が鋭い」形状。これが一目で「KAGAMIのサイトだ」と分かるブランド識別子になっている。

AI生成LPは `border-radius: 12px` をどこにでも使い、何の特徴もない。

#### 原則2: 色の「純度」の統一

ページ全体がブルーの濃淡だけで構成されている。暖色を一切混ぜない。これにより、見た目の「まとまり感」が強烈に出る。AI生成LPは「アクセントカラー」として異なる色相を追加しがちで、結果的にまとまりがなくなる。

#### 原則3: 半透明レイヤーによる奥行き

`rgba(255,255,255,0.5)`, `0.6`, `0.698`, `0.7` と微妙に異なる不透明度の白を重ね、ガラス的な奥行きを作っている。AI生成LPは要素を `background: white` で塗りつぶし、全てが同一平面上に見える。

## 12. 実装の注意点（UZOU LP への応用）

### 移植可能な技法

1. **非対称角丸の体系**: `8px 40px` のような独自パターンを定義し、全カードに一貫適用
2. **有彩色影**: 影色をブランドカラーのRGBAにする（blur 50pxのグロー影）
3. **半透明レイヤーの重なり**: カードの外枠・内枠で異なるopacityの白を使う
4. **左揃え基調のテキスト**: ヒーロー以外は中央揃えを制限的に使う
5. **マージンの緩急**: 大転換点で大きなマージン（180-196px）、連続セクションは0px
6. **テキスト/グラフィックの左右反転**: 隣接セクションでレイアウトを交互にする
7. **ドットアイコンの統一**: セクション見出しとボタンに同じ構造のアイコンを使う

### UZOUのコンテキストに合わせた変更

- KAGAMIのブルー単色は教育DXの信頼感に適合。UZOUの場合はデジタル広告プラットフォームとして異なる色相体系が必要
- KAGAMIはSTUDIO製のためアニメーションは最小限。UZOUはカスタム実装なので、より表現力のあるモーションを追加できる
- KAGAMIはBtoB教育サービスなので保守的。UZOUのターゲットに合わせたトーン調整が必要
